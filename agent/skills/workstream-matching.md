# Workstream Matching

## When to Use
When a member says they want to contribute, help, build, or get involved — and you need to match them to the right workstream.

## Procedure
1. Ask ONE question: "What kind of work energizes you — building tools, researching policy, organizing people, or something else?"
2. Based on their answer, recommend ONE primary workstream:
   - **Building tools / coding / design** → Governance Toolkit or Assembly Platform
   - **Research / policy / analysis** → AI Accountability Dashboard
   - **Organizing / events / community** → Local Hub Infrastructure
   - **Writing / communication / outreach** → Any workstream's communication role
3. Give a ONE-SENTENCE description of why that workstream fits them
4. Offer a concrete next step: "The first thing you could do is..."
5. Only mention other workstreams if they ask

## Pitfalls
- Do NOT list all four workstreams. Pick the best fit.
- Do NOT explain what a workstream is if they already know.
- Do NOT say "all workstreams need help." That is true and useless.
